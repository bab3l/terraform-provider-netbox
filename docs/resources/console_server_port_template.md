# netbox_console_server_port_template (Resource)

Manages a console server port template in NetBox. Console server port templates define the default console server ports that will be created on new devices of a specific device type or modules of a module type.

## Example Usage

```terraform
resource "netbox_manufacturer" "test" {
  name = "Test Manufacturer"
  slug = "test-manufacturer"
}

resource "netbox_device_type" "test" {
  model        = "Test Model"
  slug         = "test-model"
  manufacturer = netbox_manufacturer.test.id
  u_height     = 1
}

resource "netbox_console_server_port_template" "test" {
  name        = "Console Server Port Template"
  device_type = netbox_device_type.test.model
  type        = "rj-45"

  # Partial custom fields management
  # Only specified custom fields are managed, others preserved
  custom_fields = [
    {
      name  = "access_method"
      value = "ssh"
    },
    {
      name  = "baud_rate"
      value = "9600"
    }
  ]
}

# Optional: seed owned custom fields during import
import {
  to = netbox_console_server_port_template.test
  id = "123"

  identity = {
    custom_fields = [
      "access_method:text",
      "baud_rate:integer",
    ]
  }
}
```

## Schema

### Required

- `name` (String) The name of the console server port template. Use {module} as a substitution for the module bay position when attached to a module type.

### Optional

- `description` (String) Description of the console server port template.
- `device_type` (String) The device type ID or slug. Either device_type or module_type must be specified.
- `label` (String) Physical label of the console server port template.
- `module_type` (String) The module type ID or model name. Either device_type or module_type must be specified.
- `type` (String) The type of console server port (e.g., de-9, db-25, rj-45, usb-a, usb-b, usb-c, usb-mini-a, usb-mini-b, usb-micro-a, usb-micro-b, usb-micro-ab, other).

### Read-Only

- `id` (Number) The unique numeric ID of the console server port template.

## Import

Import is supported using the following syntax:

```shell
# Console Server Port Templates can be imported by ID
terraform import netbox_console_server_port_template.test 123
```
