# netbox_custom_field (Resource)

Manages a custom field in NetBox. Custom fields allow extending NetBox objects with additional user-defined attributes.

## Example Usage

```terraform
resource "netbox_custom_field" "test" {
  name          = "test_field"
  content_types = ["dcim.device"]
  type          = "text"
  label         = "Test Field"
  required      = false
}
```

## Schema

### Required

- `name` (String) Internal field name. Must be lowercase and contain only letters, numbers, and underscores.
- `object_types` (Set of String) The object types this custom field applies to (e.g., 'dcim.device', 'virtualization.virtualmachine').
- `type` (String) The type of custom field. Valid values: `text`, `longtext`, `integer`, `decimal`, `boolean`, `date`, `datetime`, `url`, `json`, `select`, `multiselect`, `object`, `multiobject`.

### Optional

- `choice_set` (String) The choice set name for select and multiselect custom fields.
- `comments` (String) Additional comments or notes about the custom field. Supports Markdown formatting.
- `default` (String) Default value for the field (must be a JSON value). Encapsulate strings with double quotes.
- `description` (String) Description of the custom field.
- `filter_logic` (String) Filter logic for the custom field. Valid values: `disabled`, `loose`, `exact`.
- `group_name` (String) Custom fields within the same group will be displayed together.
- `is_cloneable` (Boolean) Replicate this value when cloning objects.
- `label` (String) Name of the field as displayed to users. If not provided, the field's name will be used.
- `related_object_type` (String) The related object type for object and multiobject custom fields (e.g., 'dcim.device').
- `required` (Boolean) If true, this field is required when creating new objects or editing an existing object.
- `search_weight` (Number) Weighting for search. Lower values are considered more important. Fields with a search weight of zero will be ignored.
- `ui_editable` (String) UI editability setting. Valid values: `yes`, `no`, `hidden`.
- `ui_visible` (String) UI visibility setting. Valid values: `always`, `if-set`, `hidden`.
- `validation_maximum` (Number) Maximum allowed value (for numeric fields).
- `validation_minimum` (Number) Minimum allowed value (for numeric fields).
- `validation_regex` (String) Regular expression to enforce on text field values. Use ^ and $ to force matching of entire string.
- `weight` (Number) Fields with higher weights appear lower in a form.

### Read-Only

- `id` (String) The unique numeric ID of the custom field.
