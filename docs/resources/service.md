# netbox_service (Resource)

Manages a network service in NetBox. Services represent TCP/UDP services running on devices or virtual machines.

## Example Usage

```terraform
resource "netbox_site" "test" {
  name = "Test Site"
  slug = "test-site"
}

resource "netbox_device_role" "test" {
  name    = "Test Role"
  slug    = "test-role"
  color   = "ff0000"
  vm_role = false
}

resource "netbox_manufacturer" "test" {
  name = "Test Manufacturer"
  slug = "test-manufacturer"
}

resource "netbox_device_type" "test" {
  model        = "Test Model"
  slug         = "test-model"
  manufacturer = netbox_manufacturer.test.slug
  u_height     = 1
}

resource "netbox_device" "test" {
  name        = "Test Device"
  device_type = netbox_device_type.test.model
  role        = netbox_device_role.test.slug
  site        = netbox_site.test.slug
}

resource "netbox_service" "test" {
  name     = "SSH"
  protocol = "tcp"
  ports    = [22]
  device   = netbox_device.test.name
}
```

## Schema

### Required

- `name` (String) The name of the service (e.g., 'ssh', 'http', 'https').
- `ports` (List of Number) List of port numbers the service listens on.
- `protocol` (String) The protocol used by the service. Valid values: `tcp`, `udp`, `sctp`.

### Optional

- `comments` (String) Additional comments or notes about the service. Supports Markdown formatting.
- `custom_fields` (Attributes Set) Custom fields assigned to this resource. Custom fields must be defined in Netbox before use. (see [below for nested schema](#nestedatt--custom_fields))
- `description` (String) Description of the service.
- `device` (String) The device this service runs on (ID or name). Mutually exclusive with virtual_machine.
- `ipaddresses` (List of Number) List of IP address IDs associated with this service.
- `tags` (Attributes Set) Tags assigned to this resource. Tags must already exist in Netbox. (see [below for nested schema](#nestedatt--tags))
- `virtual_machine` (String) The virtual machine this service runs on (ID or name). Mutually exclusive with device.

### Read-Only

- `id` (String) The unique numeric ID of the service.
