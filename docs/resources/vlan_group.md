# netbox_vlan_group (Resource)

Manages a VLAN Group in Netbox. VLAN Groups are used to organize VLANs and ensure VLAN ID uniqueness within a specific scope such as a site, location, or region.

## Example Usage

```terraform
resource "netbox_site" "test" {
  name   = "Test Site"
  slug   = "test-site"
  status = "active"
}

resource "netbox_vlan_group" "test" {
  name        = "Test VLAN Group"
  slug        = "test-vlan-group"
  scope_type  = "dcim.site"
  scope_id    = netbox_site.test.id
  description = "Primary VLAN group for datacenter"

  # Partial custom fields management
  # Only specified custom fields are managed, others preserved
  custom_fields = [
    {
      name  = "vlan_range"
      value = "100-199"
    },
    {
      name  = "group_purpose"
      value = "production"
    },
    {
      name  = "max_vlans"
      value = "100"
    }
  ]

  tags = [
    "vlan-group",
    "datacenter"
  ]
}

# Optional: seed owned custom fields during import
import {
  to = netbox_vlan_group.test
  id = "123"

  identity = {
    custom_fields = [
      "vlan_range:text",
      "group_purpose:text",
      "max_vlans:integer",
    ]
  }
}
```

### Required

- `name` (String) Name of the VLAN Group.
- `slug` (String) URL-friendly identifier for the VLAN Group. Must be unique and contain only lowercase letters, numbers, hyphens, and underscores.

### Optional

- `custom_fields` (Attributes Set) Custom fields assigned to this resource. Custom fields must be defined in Netbox before use. (see [below for nested schema](#nestedatt--custom_fields))
- `description` (String) Description of the VLAN Group.
- `scope_id` (String) The ID of the object to scope this VLAN Group to. Must be used together with `scope_type`.
- `scope_type` (String) The type of object to scope this VLAN Group to. Valid values: `dcim.site`, `dcim.sitegroup`, `dcim.region`, `dcim.location`, `dcim.rack`, `virtualization.clustergroup`, `virtualization.cluster`.
- `tags` (Set of String) Tags assigned to this resource. Tags must already exist in Netbox.
