# netbox_circuit_type (Resource)

Manages a circuit type in Netbox. Circuit types categorize the various types of circuits used by your organization (e.g., Internet Transit, MPLS, Point-to-Point, Metro Ethernet, etc.).

## Example Usage

```terraform
resource "netbox_circuit_type" "test" {
  name = "Internet Transit"
  slug = "internet-transit"
}
```

## Schema

### Required

- `name` (String) The name of the circuit type. This should be descriptive and human-readable (e.g., 'Internet Transit', 'MPLS VPN', 'Dark Fiber').
- `slug` (String) The URL-friendly slug for the circuit type. Must contain only lowercase letters, numbers, and hyphens.

### Optional

- `color` (String) The color to use when displaying this circuit type (6-character hex code without the leading #, e.g., 'aa1409').
- `custom_fields` (Attributes Set) Custom fields assigned to this resource. Custom fields must be defined in Netbox before use. (see [below for nested schema](#nestedatt--custom_fields))
- `description` (String) A description of the circuit type.
- `tags` (Attributes Set) Tags assigned to this resource. Tags must already exist in Netbox. (see [below for nested schema](#nestedatt--tags))

### Read-Only

- `id` (String) The unique numeric ID of the circuit type.
