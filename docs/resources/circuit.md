# netbox_circuit (Resource)

Manages a circuit in Netbox. Circuits represent physical or logical network connections provided by external carriers or service providers.

## Example Usage

```terraform
resource "netbox_provider" "test" {
  name = "Test Provider"
  slug = "test-provider"
}

resource "netbox_circuit_type" "test" {
  name = "Internet Transit"
  slug = "internet-transit"
}

resource "netbox_circuit" "test" {
  cid              = "CID-12345"
  circuit_provider = netbox_provider.test.id
  type             = netbox_circuit_type.test.id
  status           = "active"
  description      = "Main Internet Circuit"
}
```

## Schema

### Required

- `cid` (String) The unique circuit ID assigned by the provider. This is typically a service order number or circuit identifier from the carrier.
- `circuit_provider` (String) The circuit provider (carrier or ISP) supplying this circuit. Can be specified by name, slug, or ID.
- `type` (String) The type of circuit (e.g., Internet Transit, MPLS, Point-to-Point). Can be specified by name, slug, or ID.

### Optional

- `comments` (String) Additional comments or notes about the circuit.
- `commit_rate` (Number) The committed information rate (CIR) in Kbps for this circuit.
- `custom_fields` (Attributes Set) Custom fields assigned to this resource. Custom fields must be defined in Netbox before use. (see [below for nested schema](#nestedatt--custom_fields))
- `description` (String) A description of the circuit.
- `install_date` (String) The date when the circuit was installed, in YYYY-MM-DD format.
- `status` (String) The operational status of the circuit. Valid values are: `planned`, `provisioning`, `active`, `offline`, `deprovisioning`, `decommissioned`. Defaults to `active`.
- `tags` (Attributes Set) Tags assigned to this resource. Tags must already exist in Netbox. (see [below for nested schema](#nestedatt--tags))
- `tenant` (String) The tenant that owns this circuit. Can be specified by name, slug, or ID.
- `termination_date` (String) The date when the circuit will be or was terminated, in YYYY-MM-DD format.

### Read-Only

- `id` (String) The unique numeric ID of the circuit.
