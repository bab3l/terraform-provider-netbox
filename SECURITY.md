# Security Policy

## Supported Versions
We aim to support the latest minor versions of the provider. Report issues against the current `main` branch or the latest release.

## Reporting a Vulnerability
- Please report security issues privately.
- Email: security@bab3l.dev (or open a private security advisory if enabled)
- Include:
  - Affected version(s)
  - Reproduction steps
  - Impact assessment

We will triage within 5 business days and provide a remediation or timeline.

## Guidelines
- Do not disclose vulnerabilities publicly until a fix is available.
- Avoid submitting secrets/tokens in reports.
- For dependency-related CVEs, we update via Dependabot and release patches promptly.
